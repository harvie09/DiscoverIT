Java Language

Lesson 1

What is Java?

A full-fledged programming language in which you can accomplish the same sorts of tasks and solve the same sorts of problems that you can do in other programming languages.



History of Java

It was originally developed by Sun Microsystems. James Gosling initiated Java language project in June 1991 for the use in one of his many setup box projects.


Sun released the first public implementation as Java 1.0 in 1995 as Write Once, Run Anywhere(WORA). From GreenTalk as a .gt file it was revised to Oak.


Oak was renamed as "Java" because it was already a trademark by Oak Technologies.


On November 13, 2006, Sun released much of Java as free and open-source software under the terms of the GNU General Public License (GPL).


On May 8, 2007, Sun finished the process, making all of Java's core code free and open-source.

Why Learn Java?

Simple - Java is designed to be easy to learn.
Dynamic - It is suitable for any programming task and allows us to build an application for everyone.
In-demand - It has remained one of the most in-demand programming languages in the world.

Characteristic of Java

Simple - Java removes several C/C++ features considered to be unsafe such as pointer arithmetic, struct, typedefs etc. and the need to free memory.

Object-Oriented - It enforces OOP design from the start.

Robust - Java works hard to check for problems at compile time and run time.

Secure - Bytecode verifies checks for illegal codes.

Distributed - Java has extensive TCP/IP capabilities that have built in, which allow easy access to information over a network.

Portable - Java implement other portability standards.

Fast - The Java interpreter can be relatively slow in executing but the JIT compilers can speed up.

Multithreaded - Java allows many simultaneous activities in one program via the Thread class.

Java Applications

Typically compiled to byte code (class file) that can run any Java Virtual Machine (JVM).
Some devices that are using Java:
1. Desktop Application such as acrobat reader, media play, antivirus, etc.
2. Web Applications
3. Enterprise Applications such as banking applications.
4. Mobile
5. Embedded System
6. Smart Card
7. Robotics
8. Games, etc.

Types of Java Application


1. Standalone Application


Standalone applications are also known desktop applications or window-based applications. These are traditional software that we need to install on every machine.



2. Web Application


An application that runs on the server side and creates a dynamic page is called a web application.
Currently, Servlet, JSP, Struts, Spring, Hibernate, JSF, etc. technologies are used for creating web applications in Java.



3. Enterprise Application


An application that is distributed in nature, such as banking applications, etc. is called enterprise application.

It has advantages of the high-level security, load balancing, and clustering. In Java, EJB is used for creating enterprise applications.



4. Mobile Application


An application which is created for mobile devices is called a mobile application. Currently, Android and Java ME are used for creating mobile applications.

Rules and Syntax

1. Case Sensitivity - Java is case sensitive, which means identifier Hello and hello would have different meaning in Java.
2. Class Names - For all class names the first letter should be in Upper Case. If several words are used to form a name of the class, each inner word's first letter should be in Upper Case.

Pascal Naming Convention

ex: HelloWorld, GoodbyeWorld


Valid Class Names in Java

3. Program File Name - Name of the program file should exactly match the class name. When saving the file, you shoould save it using the class name and append '.java' to the end of the name.

Example


-MyFirstJavaProgram (Class Name)
-MyFirstJavaProgram.java (File Name)



4. public static void main(String args[]) - Java program processing starts from the main() method which is a mandatory part of every Java program.


5. Method Names - All method names should start with a lower-case letter. If several words are used to form the name of the method, then each inner word's first letter should be in Upper case.

Camel Naming Convention

ex: firstFunction, gradeCalculator
